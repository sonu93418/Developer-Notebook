What is JavaScript?

JavaScript is a versatile, high-level programming language that powers interactive web experiences. Originally created for browsers, JavaScript has evolved into a full-stack development language used on servers, mobile applications, and desktop applications.

Key Features

  • Dynamic Typing: Variables can hold any data type and change types dynamically
  • First-Class Functions: Functions are objects that can be passed around like any other value
  • Event-Driven: Perfect for responding to user interactions
  • Asynchronous Support: Callbacks, Promises, and async/await for non-blocking operations
  • Prototype-Based: Object-oriented programming through prototypes instead of classes

Browser APIs

JavaScript interacts with the browser through powerful APIs:

// DOM Manipulation
document.getElementById('id').textContent = 'Hello';

// Event Handling
element.addEventListener('click', () => {
  console.log('Clicked!');
});

// Fetch API
fetch('/api/data')
  .then(response => response.json())
  .then(data => console.log(data));

Frontend Frameworks

JavaScript powers major frontend frameworks:

  • React: Component-based UI library
  • Vue: Progressive JavaScript framework
  • Angular: Full-featured web framework
  • Svelte: Compiler-based framework

Node.js & Backend

With Node.js, JavaScript can run on servers:

// Express.js Server
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello from Node.js!');
});

app.listen(3000);

Best Practices

  • Use const and let instead of var
  • Handle errors with try/catch or .catch()
  • Use meaningful variable and function names
  • Keep functions small and focused
  • Use arrow functions for concise syntax

Object-Oriented Programming

// ES6 Class
class User {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    return `Hello, I'm ${this.name}`;
  }

  isAdult() {
    return this.age >= 18;
  }
}

const user = new User('Alice', 25);
console.log(user.greet()); // Hello, I'm Alice

Advanced Patterns

  • Module Pattern: Encapsulate code in functions to prevent global pollution
  • Observer Pattern: Implement event-driven architecture
  • Middleware Pattern: Chain operations for request processing
  • Factory Pattern: Create objects without specifying exact classes
  • Closure Pattern: Maintain private state and functions

Performance Optimization

// Debounce expensive operations
const debounce = (fn, delay) => {
  let timeoutId;
  return (...args) => {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => fn(...args), delay);
  };
};

// Throttle repeated events
const throttle = (fn, delay) => {
  let lastCall = 0;
  return (...args) => {
    const now = Date.now();
    if (now - lastCall >= delay) {
      fn(...args);
      lastCall = now;
    }
  };
};

Common Use Cases
