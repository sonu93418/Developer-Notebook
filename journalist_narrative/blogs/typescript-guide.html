What is TypeScript?

TypeScript is a strongly-typed superset of JavaScript that compiles to plain JavaScript. It adds static type checking, interfaces, and other features that make code more robust, maintainable, and easier to debug.

Core Features

  • Static Types: Catch errors at compile-time instead of runtime
  • Interfaces: Define contracts for object structures
  • Classes: Full object-oriented programming support
  • Generics: Write reusable, flexible components
  • Decorators: Add metadata and modify classes and methods

Type Annotations

TypeScript allows you to specify types for variables, parameters, and return values:

// Variable Types
const name: string = 'John';
const age: number = 30;
const active: boolean = true;

// Function Types
function add(a: number, b: number): number {
  return a + b;
}

// Union Types
let value: string | number;
value = 'hello'; // OK
value = 42;      // OK

Interfaces

Define the structure of objects with interfaces:

interface User {
  id: number;
  name: string;
  email: string;
  isActive?: boolean; // Optional property
}

const user: User = {
  id: 1,
  name: 'John',
  email: 'john@example.com',
  isActive: true
};

Classes & Inheritance

class Animal {
  name: string;
  
  constructor(name: string) {
    this.name = name;
  }
  
  speak() {
    console.log(`${this.name} makes a sound`);
  }
}

class Dog extends Animal {
  speak() {
    console.log(`${this.name} barks`);
  }
}

Generics

Create flexible, reusable components that work with multiple types:

// Generic Function
function identity<T>(arg: T): T {
  return arg;
}

// Generic Array
const items: Array<string> = ['a', 'b', 'c'];

// Generic Interface
interface Container<T> {
  value: T;
  getValue(): T;
}

Why Use TypeScript?

"TypeScript brings the benefits of static typing to JavaScript. Better IDE support, fewer runtime errors, and code that's easier to maintain and refactor."

Benefits in Teams

  • Documentation: Types serve as self-documenting code
  • IDE Support: Better autocomplete and refactoring
  • Error Prevention: Catch mistakes before runtime
  • Collaboration: Clear contracts between modules
  • Maintenance: Easier to understand code intent

Advanced Type Features

// Conditional types
type IsString<T> = T extends string ? true : false;

// Utility types
type Readonly<T> = { readonly [K in keyof T]: T[K] };
type Partial<T> = { [K in keyof T]?: T[K] };

// Discriminated unions
type Result<T> = 
  | { success: true; value: T }
  | { success: false; error: string };

Common Use Cases

  • Large Codebases: Catch type errors before production
  • Team Projects: Self-documenting code through types
  • APIs & Libraries: Publish type-safe packages
  • Full-Stack Development: Share types between client and server
  • Long-term Maintenance: Easier refactoring and updates

Decorators & Metadata

// Using decorators (with experimental flag)
@sealed
class User {
  @validate
  name: string;
}

function sealed(target: Function) {
  Object.seal(target);
}

Testing & Quality

// Jest with TypeScript
describe('UserService', () => {
  it('creates user correctly', () => {
    const user: User = { 
      id: 1, 
      name: 'Alice' 
    };
    expect(user.name).toBe('Alice');
  });
});
