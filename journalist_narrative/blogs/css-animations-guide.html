Introduction to CSS Animations

CSS animations bring life to your web pages without requiring any JavaScript. They're smooth, performant, and provide an excellent user experience when used effectively. Whether you're creating subtle transitions or eye-catching effects, CSS has you covered.

Transitions vs Animations

There's an important distinction between CSS transitions and animations:

Transitions

Transitions smoothly change property values over time in response to an event:

.button {
  background: #007bff;
  padding: 10px 20px;
  transition: background 0.3s ease;
}

.button:hover {
  background: #0056b3;
}

Animations

Animations use keyframes to define multiple steps in the animation sequence:

@keyframes slide {
  0% {
    transform: translateX(0);
  }
  50% {
    transform: translateX(50px);
  }
  100% {
    transform: translateX(100px);
  }
}

.element {
  animation: slide 2s ease-in-out infinite;
}

Example: Sliding animation

Keyframes Explained

Keyframes define what styles should be applied at different points in the animation. You can specify them using percentages or keywords (from/to):

@keyframes fadeInSlide {
  0% {
    opacity: 0;
    transform: translateY(-20px);
  }
  50% {
    opacity: 0.5;
  }
  100% {
    opacity: 1;
    transform: translateY(0);
  }
}

Animation Properties

Control your animations with these properties:

  • animation-name - The name of the keyframes animation
  • animation-duration - How long the animation takes to complete
  • animation-timing-function - How the animation progresses (ease, linear, cubic-bezier)
  • animation-delay - Delay before animation starts
  • animation-iteration-count - Number of times to repeat (or infinite)
  • animation-direction - Play forward, backward, or alternate
  • animation-fill-mode - How styles are applied outside animation time

Example: Rotating animation

Timing Functions

Timing functions control the animation's acceleration and deceleration:

/* Common timing functions */
ease           /* Starts slow, fast in middle, ends slow */
linear         /* Constant speed throughout */
ease-in        /* Starts slow, accelerates */
ease-out       /* Starts fast, decelerates */
ease-in-out    /* Slow start and end */

/* Custom timing */
cubic-bezier(0.25, 0.1, 0.25, 1)

Example: Fading animation

Performance Tips

Create smooth, performant animations by following these best practices:

  • Use transform and opacity: These properties are GPU-accelerated and perform best
  • Avoid animating: width, height, left, right, top, bottom (use transform instead)
  • Use will-change sparingly: Only on elements that are actually animated
  • Test on mobile: Mobile devices have limited resources
  • Reduce motion: Respect user's prefers-reduced-motion preference

Respecting User Preferences

Always respect the user's animation preferences:

@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}

Real-World Examples

Fade In on Load

@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

body {
  animation: fadeIn 1s ease-out;
}

Loading Spinner

@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

.spinner {
  animation: spin 1s linear infinite;
}

Hover Effect

.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.card:hover {
  transform: translateY(-5px);
  box-shadow: 0 10px 20px rgba(0, 0, 0, 0.2);
}

Conclusion

CSS animations are a powerful tool for enhancing user experience. By understanding keyframes, timing functions, and performance best practices, you can create smooth, engaging animations that don't compromise performance.
