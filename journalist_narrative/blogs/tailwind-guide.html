What is Tailwind CSS?

Tailwind CSS is a utility-first CSS framework for rapidly building custom user interfaces. Instead of writing custom CSS, you use pre-defined utility classes directly in your HTML to build components and layouts. This approach leads to faster development and more maintainable code.

Why Tailwind?

  • Utility-First: Build designs without leaving your HTML
  • Fast Development: No need to name classes or write custom CSS
  • Consistency: Built-in design system with colors, spacing, fonts
  • Responsive: Mobile-first breakpoints built-in
  • Customizable: Extend or override the default theme

Installation

// Using npm
npm install -D tailwindcss postcss autoprefixer

// Generate config files
npx tailwindcss init -p

// In tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{html,js}",

  theme: {
    extend: {},
  },

Advanced Features

  • Dark Mode: Built-in dark mode support with dark: prefix
  • Animations: Pre-built animations like pulse, bounce, spin
  • Plugins: Extend Tailwind with custom functionality

  Content that changes with dark mode

<!-- Enable dark mode in tailwind.config.js -->
module.exports = {

}

Creating Reusable Components

/* Using @apply in CSS */
@layer components {
  .btn-primary {
    @apply px-4 py-2 bg-blue-500 text-white rounded-lg;
    @apply hover:bg-blue-600 transition-colors;
  }

  .card {
    @apply bg-white p-6 rounded-lg shadow-lg;
  }
}

/* In HTML */
<button class="btn-primary">Click me</button>
<div class="card">Card content</div>

Plugin Development

// tailwind.config.js
module.exports = {
  plugins: [
    function({ addComponents, theme }) {
      addComponents({
        '.btn-custom': {
          padding: theme('spacing.3'),
          borderRadius: theme('borderRadius.lg'),
          fontWeight: theme('fontWeight.bold'),
        },
      })
    }
  ]
}

State Variants

<!-- Hover, focus, active states -->
<button class="bg-blue-500 hover:bg-blue-600 focus:outline-none 
  focus:ring-2 active:scale-95">
  Interactive Button

<!-- Group hover for parent-child interactions -->
<div class="group border hover:bg-blue-100">
  <h3 class="group-hover:text-blue-600">Title</h3>
  <p class="group-hover:text-blue-500">Subtitle</p>
</div>

Best Practices

"Tailwind CSS empowers developers to build beautiful, responsive interfaces faster than ever. Learn it once, and you'll wonder how you ever built sites without it."
  • Use responsive modifiers for mobile-first design (sm:, md:, lg:, xl:, 2xl:)
  • Leverage the @apply directive to avoid repetitive classes
  • Customize the theme to match your brand identity
  • Extract components using @layer components for reusability
  • Purge unused CSS in production for optimal bundle size
  • Use state variants (hover:, focus:, active:) for interactivity
  • Maintain a consistent design system across your project
